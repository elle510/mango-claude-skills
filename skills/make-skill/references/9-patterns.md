# 스킬 유형 9가지

좋은 스킬은 아래 유형 중 **하나에만** 깔끔하게 들어맞는다.
여러 유형에 걸쳐 있으면 스킬을 쪼개거나 목적을 재정의해야 한다.

## 체크리스트

| #   | 유형                    | 핵심 질문                                             | 예시                                       |
| --- | ----------------------- | ----------------------------------------------------- | ------------------------------------------ |
| 1   | **라이브러리 레퍼런스** | 우리 내부 라이브러리/API를 어떻게 쓰는가?             | `use-our-sdk`, `query-conventions`         |
| 2   | **제품 검증**           | 코드 변경 후 실제 동작을 자동으로 검증하는가?         | Playwright 연동 테스트, 스냅샷 비교        |
| 3   | **데이터 조회**         | 모니터링/분석 스택에서 데이터를 꺼내오는가?           | Grafana, BigQuery, DataDog 연동            |
| 4   | **비즈니스 자동화**     | 반복되는 문서/보고서 작업을 자동화하는가?             | 스탠드업, 주간 리포트, 릴리즈 노트         |
| 5   | **코드 스캐폴딩**       | 새 파일/컴포넌트/서비스를 정해진 패턴으로 생성하는가? | `make-feature`, `make-entity`, `make-page` |
| 6   | **코드 리뷰**           | 코드 스타일이나 아키텍처 규칙을 검사하고 강제하는가?  | `fsd-review`, `security-review`            |
| 7   | **CI/CD**               | PR 상태 모니터링, 배포, 롤백을 다루는가?              | `deploy`, `rollback`, `watch-pr`           |
| 8   | **런북(Runbook)**       | 장애 알림 시 자동으로 조사하고 보고서를 만드는가?     | `on-call`, `incident-report`               |
| 9   | **인프라 운영**         | 리소스 정리, 비용 분석, 환경 관리를 하는가?           | `cleanup-stale`, `cost-report`             |

## 판단 방법

1. 만들려는 스킬이 "무엇을 하는가"를 한 문장으로 적는다.
2. 위 표에서 가장 잘 맞는 유형 하나를 고른다.
3. 두 유형에 걸쳐 있다면 → 스킬을 두 개로 분리하거나, 더 좁은 범위로 재정의한다.

## 이 레포지토리의 스킬 유형

| 스킬              | 유형                          |
| ----------------- | ----------------------------- |
| `commit`          | 비즈니스 자동화 (커밋 메시지) |
| `claudemd-review` | 코드 리뷰                     |
| `make-skill`      | 코드 스캐폴딩                 |

## 누락된 유형 점검

위 표를 보고 팀에서 아직 스킬로 만들지 않은 유형이 있는지 확인한다.
특히 **제품 검증(2번)** 유형은 가장 높은 ROI를 가지므로 우선순위가 높다.
